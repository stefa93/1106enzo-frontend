# Design Tokens

Our design system uses Tailwind CSS for consistent styling across components. These tokens are defined in `tailwind.config.ts`.

## Colors

```tsx
colors: {
  primary: {
    DEFAULT: 'hsl(var(--primary))',
    foreground: 'hsl(var(--primary-foreground))',
  },
  secondary: {
    DEFAULT: 'hsl(var(--secondary))',
    foreground: 'hsl(var(--secondary-foreground))',
  },
  accent: {
    DEFAULT: 'hsl(var(--accent))',
    foreground: 'hsl(var(--accent-foreground))',
  },
}
```

Usage:
```tsx
<button className="bg-primary text-primary-foreground">
  Click me
</button>
```

## Typography

Font families and sizes are configured in Tailwind:

```tsx
fontSize: {
  xs: ['0.75rem', { lineHeight: '1rem' }],
  sm: ['0.875rem', { lineHeight: '1.25rem' }],
  base: ['1rem', { lineHeight: '1.5rem' }],
  lg: ['1.125rem', { lineHeight: '1.75rem' }],
  xl: ['1.25rem', { lineHeight: '1.75rem' }],
}
```

Usage:
```tsx
<h1 className="text-2xl font-bold">
  Heading
</h1>
```

## Spacing

Consistent spacing using Tailwind's scale:

```tsx
spacing: {
  0: '0',
  1: '0.25rem',
  2: '0.5rem',
  3: '0.75rem',
  4: '1rem',
  // ...
}
```

Usage:
```tsx
<div className="p-4 m-2 space-y-4">
  Content
</div>
```

## Breakpoints

Responsive design breakpoints:

```tsx
screens: {
  sm: '640px',
  md: '768px',
  lg: '1024px',
  xl: '1280px',
  '2xl': '1536px',
}
```

Usage:
```tsx
<div className="w-full md:w-1/2 lg:w-1/3">
  Responsive content
</div>
```

## Dark Mode

Support for dark mode using Tailwind's `dark:` modifier:

```tsx
<div className="bg-white text-gray-900 dark:bg-gray-800 dark:text-gray-100">
  Dark mode aware content
</div>
```

## Component Examples

### Button Variants

```tsx
const variants = {
  primary: 'bg-primary text-primary-foreground hover:bg-primary/90',
  secondary: 'bg-secondary text-secondary-foreground hover:bg-secondary/80',
  ghost: 'hover:bg-accent hover:text-accent-foreground',
};
```

### Spacing Patterns

```tsx
<div className="space-y-4 p-6">
  <h2 className="text-xl font-semibold">Section Title</h2>
  <p className="text-gray-600 dark:text-gray-300">Content</p>
</div>
```

## Best Practices

1. Use semantic color tokens instead of raw colors
2. Follow spacing scale for consistency
3. Design mobile-first using breakpoints
4. Support dark mode for all components
5. Use typography scale for font sizes

## Usage Example

```tsx
<div className="bg-blue-600 text-white p-4 rounded-lg shadow-md transition-all duration-200 hover:bg-blue-700">
  <h2 className="text-xl font-semibold mb-2">Example Component</h2>
  <p className="text-sm">Using our design tokens</p>
</div>
```
