## Version Management

We follow [Semantic Versioning](https://semver.org/):
- MAJOR version for incompatible API changes
- MINOR version for new functionality in a backwards compatible manner
- PATCH version for backwards compatible bug fixes

## Release Process

1. Components start in experimental phase
2. After testing and review, marked as stable
3. Breaking changes trigger major version bump
4. Deprecations announced one major version ahead
